Git Commands and Concepts

Generating SSH Key Pair

Command: ssh-keygen -t rsa -C "<emailid>"

After generating the SSH key pair, add the public key to your GitHub account in settings.

Configuring User Information

Commands:

  • git config --global user.name "username"
  • git config --global user.email "emailid"

Checking Remote Repositories

Command: git remote -v

Cloning a Repository

Command: git clone "<copy paste the ssh related url>"

Staging and Committing Changes

Commands:

  • git add .
  • git commit -m "message"

By committing the changes, the current version will be available even if you modify it later. The committed ones will redirect to the remote repository if we do the push command and vice-versa.

Checking Status and Logs

Commands:

  • git status
  • git log

Pushing and Pulling Changes

Commands:

  • git push
  • git pull

GUI Tools

Commands:

  • git gui&
  • gitk&

Version Tree

Commands:

  • git show -s --pretty=raw <SHA-1 hash>
  • git ls-tree <tree sha-1 hash>
  • git show <blob sha-1 hash>

The commits store the version as a tree, and the trees store the information in large binary object files.

Undoing Changes

From Working Directory:

  • git checkout --<file> to remove changes from that particular file
  • git checkout . to remove all changes
  • git clean -xdf to remove new files (f-force, d-directory)

From Staging Area to Working Directory:

  • git reset -- <file.txt>

From Local Repo to Staging Area:

  • git reset HEAD~n to revert the commit (n represents how many commits you want to revert)
  • git commit --amend -m "commit message" to save the changes in a file in the previous version other than creating a new one
  • git reset --soft HEAD~1 to revert the commit to the staging area
  • git reset --mixed HEAD~1 to revert the commit to the working area
  • git reset --hard HEAD~1 to remove the commit

From Remote Repo:

  • git revert <sha1>

Git Ignore

Create a .gitignore file in the repository and write the names of the files which should be ignored.

Examples:

  • *.log to ignore all .log files
  • !error.log to track error.log file even though you are ignoring .log files
  • /TODO to only ignore the TODO file
  • build/ to ignore all files in the build/ directory
  • doc/*.txt to ignore all .txt files only in the doc directory but not in subdirectories
  • doc/**/*.pdf to ignore all .pdf files in the doc directory and in subdirectories

Branching and Merging

Commands:

  • git checkout -b <branchname> to create a new branch
  • git checkout <branchname> to switch to that branch
  • git merge <branchname> to merge the branch into the current branch

Tips to Avoid Conflicts:

  • Short commits
  • No edits to whitespaces
  • Merge often

Commands to Resolve Conflicts:

  • git merge --abort to abort merge or cancel the merge
  • git checkout --Xours or --Xtheirs to resolve by selecting version
  • git diff to resolve merge manually
  • git revert <sha1> to undo merge

Rebase and Cherry Pick

Rebase and Cherry Pick commands are used for more advanced version control operations.

Tagging

We can name the commits using the tag command and this tag also can be used instead of SHA1.

Commands:

  • git tag <tagname>
  • git tag --list to see all the tags
  • git push --tags to push tags separately
  • git checkout <tagname> to navigate to that particular branch

Stashing

If you are doing something in your branch and you have to suddenly switch to some other branch, you can't commit the changes because they aren't complete. In such cases, stashing helps you to save the changes temporarily and switch to other branches.

Commands:

  • git stash
  • git stash list to list down the saved changes
  • git stash pop to get back the stashed changes
  • git stash apply bring them back and leave in stash
  • git stash drop{name} to remove or clear them from stash

Submodules

Sometimes you might need to use some other project as a subdirectory in your project. In such cases, you need to add a submodule.

Commands:

  • git submodule add <git_url> <directory_name> to create the submodule
  • git submodule init to update the changes
  • git submodule update to update the changes

Remote

Commands:

  • git remote -v (is to check which server we are working with)
  • git remote remove origin (to remove the existing remote)
  • git remote add <name> <url> (to add)

Extras

Commands:

  • git config --global core.editor <path>
  • git blame
  • git bisect
  • git log --pretty=oneline (to see all commits in one line)
  • git log --pretty=format:"%h %s" -graph
  • git config --global alias.last
  • git last
  • git log master..experiment
  • .git rerere
  • git submodule
  • git filter-branch --tree-filter
  • git tag -d tag-name (command where tag-name is the name of the Git tag you want to delete)
  • git tag (to list all tags)
  • git push --all origin (to push all branches)

Additional Commands

Commands:

  • echo
